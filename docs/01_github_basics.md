# GitHub Basics for This Project

## What is a repository?

A repository is your project folder on GitHub.

Example:

```text
retail-intelligence-platform
```

## What is a commit?

A commit is a saved version of your work.

Good commit examples:

```text
Add sample sales data
Create inventory KPI SQL
Update README with business objective
```

Bad commit examples:

```text
update
final
changes
```

## What is a branch?

A branch is a safe copy where you make changes before adding them to the main project.

Example branch names:

```text
add-sales-analysis
create-inventory-dashboard
fix-forecast-kpi
```

## What is a pull request?

A pull request is a review request.

It means:

> I made changes. Please review before merging into the main project.

## Beginner Workflow

1. Create or edit a file
2. Commit the change
3. Create a branch for bigger work
4. Open a pull request
5. Review and merge

## Your First GitHub Practice

Create one issue:

```text
Create sales analysis SQL
```

Then create one branch:

```text
feature/sales-analysis
```

Then edit one SQL file and commit it.
